# Source Notes: Linear Algebra and Operator Basics

These notes are not canonical graph facts. They are batch-specific extraction context for the autonomous proposer, aimed at the linear algebra and operator vocabulary reused later in QM and QFT.

## Intended Reading of the Topic

For this batch, `linear algebra and operator basics` should be understood as the shared vocabulary needed to recognize and use the basic objects and maps that appear in standard linear algebra writing:

- vector spaces and subspaces
- spans, bases, and coordinates
- linear maps and linear operators
- matrix representations of linear maps
- kernel, image, composition, identity, and inverse
- eigenvalue and eigenvector language at the introductory level

This batch is not trying to capture advanced operator theory, proof-heavy structure, or the full modern literature on functional analysis.

## What the Batch Is Trying to Capture

This batch should identify the smallest defensible prerequisite slice needed to:

- recognize the basic linear-algebraic objects named in a source,
- follow ordinary explanations of how operators act on vectors or coordinates,
- and use that vocabulary in simple reading or recall tasks.

The batch is not trying to capture:

- a derivation-first treatment of rank-nullity, spectral theorem, or canonical forms,
- numerical methods or optimization,
- or broad umbrella nodes that flatten the distinction between a linear map and its matrix representation.

## Domain Hints

Likely high-value node candidates are:

- vector spaces
- subspaces
- basis and coordinate systems
- linear maps and linear operators
- matrices as representations of linear maps
- kernel and image
- identity and inverse maps
- eigenvalues and eigenvectors
- transpose or adjoint, only if the source requires the term

Likely high-value dependency patterns:

- representation-level links from matrices to linear maps
- recognition-level links from core vector-space vocabulary into operator vocabulary
- light use-level prerequisites for reading standard introductory explanations

## Anti-Patterns

Avoid:

- broad umbrella nodes like `linear algebra` or `operator theory` unless a source clearly needs them,
- importing advanced functional analysis into this first slice,
- treating matrix arithmetic as a substitute for operator meaning,
- and moving into theorem-proving content before the basic vocabulary is in place.

## Evidence Guidance

When proposing claims, prefer evidence of these forms:

- source language that names the basic objects directly,
- explanations that move from vectors to linear maps to matrices,
- and introductory references that use kernel/image/eigen language without needing advanced derivations.

If a claim feels like it belongs to a later, deeper batch, flag it instead of asserting it here.
